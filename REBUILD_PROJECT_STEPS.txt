Overview
--------
This document explains how to rebuild the `student_mgmt_system` Django project from scratch on a Windows development machine. It assumes the project source (the repository) is located at `C:\Users\Dell\Desktop\student_mgmt_system`.

Goals
- Create a reproducible environment
- Install dependencies
- Configure settings and environment variables
- Apply migrations and create initial data
- Run the development server and run tests

Prerequisites
-------------
- Python 3.10+ (3.11 recommended). Confirm with:

```powershell
python --version
```

- Git (if cloning repository)
- Visual Studio Build Tools (for some binary packages) or prebuilt wheels
- Internet access to download packages

1) Clone or obtain repository
--------------------------------
If you don't already have the repo locally:

```powershell
cd C:\Users\Dell\Desktop
git clone <repo-url> student_mgmt_system
cd student_mgmt_system
```

2) Create and activate virtual environment (Windows)
----------------------------------------------------
Use a project-local venv (recommended):

PowerShell (recommended):

```powershell
python -m venv .venv
# On first-time use you may need to allow script execution:
# Open PowerShell as Administrator and run:
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine
.\.venv\Scripts\Activate.ps1
```

Command Prompt (cmd.exe):

```cmd
python -m venv .venv
.\.venv\Scripts\activate
```

3) Upgrade pip & install build helpers
--------------------------------------
```powershell
python -m pip install --upgrade pip setuptools wheel
```

4) Create `requirements.txt` (example)
--------------------------------------
Create a `requirements.txt` in project root (adjust versions as needed). Example content:

```
Django>=4.2,<5
Pillow>=9.0
django-environ>=0.10.0
psycopg2-binary>=2.9    # only if you use PostgreSQL
whitenoise>=6.0
django-debug-toolbar>=4.0
python-dotenv>=1.0
```

Install dependencies:

```powershell
pip install -r requirements.txt
```

If any package fails to build, install the appropriate build tools or use prebuilt wheels.

5) Ensure Pillow is available (for image fields)
------------------------------------------------
Pillow is required for `ImageField`. If you hit a build error, install the required Windows libs or use a prebuilt wheel.

6) Environment variables and `settings.py` changes
-------------------------------------------------
This project uses environment-based configuration for security and flexibility. Create a `.env` file in the project root with values like:

```
SECRET_KEY=change-me-to-a-secure-random-string
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
DATABASE_URL=sqlite:///db.sqlite3
SITE_URL=http://localhost:8000

# Email Configuration (Gmail SMTP example)
EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password
DEFAULT_FROM_EMAIL=your-email@gmail.com
```

IMPORTANT: Email Configuration
- The system uses Django's SMTP email backend to send credentials to newly created student and guardian accounts
- For Gmail: Use an "App Password" (not your regular password) - see https://support.google.com/accounts/answer/185833
- For other email providers: Update EMAIL_HOST, EMAIL_PORT, and credentials accordingly
- The DEFAULT_FROM_EMAIL should match EMAIL_HOST_USER for best results
- Test email configuration by creating a new student/guardian account

If the project uses `django-environ` or `python-dotenv`, ensure the `SMS/settings.py` reads these variables. The settings file automatically loads the `.env` file at startup.

7) Create media & static folders and set permissions
---------------------------------------------------
```powershell
mkdir media
mkdir staticfiles
```

8) Apply migrations
-------------------
Run Django migrations to create database tables:

```powershell
python manage.py makemigrations
python manage.py migrate
```

Notes for this repo:
- If you have custom migration files already (for example `courses/migrations/0008_add_coursemodule_and_attachment.py`), running `migrate` will apply them. If `migrate` complains about missing tables or conflicts, inspect the migrations and order; you may need to run `python manage.py showmigrations` to see what's applied.
- If you encounter `OperationalError: no such table: ...` for custom models, ensure migrations for that app were created and run. Use:

```powershell
python manage.py makemigrations courses
python manage.py migrate courses
```

If migration names collide or you reverted code creating models, you may need to reset migration state carefullyâ€”avoid doing this in production.

9) Create a superuser
---------------------
```powershell
python manage.py createsuperuser
```
Follow prompts to set email and password.

10) Collect static files (for production-like static serving)
-----------------------------------------------------------
```powershell
python manage.py collectstatic --noinput
```

11) Run the development server
------------------------------
```powershell
python manage.py runserver
# or to bind to all interfaces
python manage.py runserver 0.0.0.0:8000
```

Open `http://127.0.0.1:8000/` in your browser.

12) Verify common runtime issues & troubleshooting
--------------------------------------------------
- ModuleNotFoundError: No module named 'django'
  - Activate the virtualenv and run `pip install -r requirements.txt`.
- OperationalError: no such table: <app>_<model>
  - Run `python manage.py migrate` for that app.
- Image upload errors
  - Ensure `MEDIA_ROOT` exists and Django has write permissions.
- Broken templates / missing static files
  - Run `collectstatic` and check `STATICFILES_DIRS` and `STATIC_ROOT` settings.

13) Running tests
-----------------
If the repo includes tests, run:

```powershell
python manage.py test
```

16) Optional: Docker (recommended for reproducible dev)
-------------------------------------------------------
A basic `Dockerfile` for development:

```
FROM python:3.11-slim
WORKDIR /app
COPY . /app
RUN pip install --upgrade pip setuptools wheel
RUN pip install -r requirements.txt
CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]
```

And a `docker-compose.yml` to run with Postgres:

```yaml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgres://postgres:postgres@db:5432/smsdb
    depends_on:
      - db
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: smsdb
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
```

16) Recent Features & Updates
------------------------------
The project has been updated with the following improvements:

Email System (IMPORTANT)
- Student and Guardian account creation now sends welcome emails with credentials

Student Management

- When adding guardians, they receive credentials via email
- After adding a student, redirect goes to the all_students_list page

Account Creation Improvements
- Both `create_student` and `add_student_to_course` views send emails
- Both `create_guardian` and guardians created during student enrollment receive emails
- Edit student page now includes editable Student ID field

Testing the Email System
- To test locally without Gmail, temporarily change EMAIL_BACKEND in settings.py to:
  `django.core.mail.backends.console.EmailBackend` (prints to console)

- For production, use your email provider's SMTP settings
