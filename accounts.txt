Accounts app — Step-by-step implementation guide

Overview
- Purpose: manage users, student/teacher/guardian profiles, authentication, and profile signals.
- Files involved: `models.py`, `forms.py`, `views.py`, `urls.py`, `admin.py`, `signals.py`, `tests.py`.

1) Models
- Create profile models linked to Django's `User` (or a custom `AbstractUser`). Example:

from django.contrib.auth.models import User
from django.db import models

class StudentProfile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    admission_no = models.CharField(max_length=50, unique=True)
    date_of_birth = models.DateField(null=True, blank=True)
    batch = models.ForeignKey('courses.Batch', null=True, blank=True, on_delete=models.SET_NULL)

    def __str__(self):
        return f"{self.user.get_full_name()} ({self.admission_no})"

2) Signals (auto-create profile)
- Connect `post_save` to create a profile when a user is created.

from django.db.models.signals import post_save
from django.dispatch import receiver

@receiver(post_save, sender=User)
def create_user_profile(sender, instance, created, **kwargs):
    if created:
        StudentProfile.objects.create(user=instance)

3) Forms
- Provide a `UserCreationForm` subclass and profile form for edits.

from django import forms
from django.contrib.auth.forms import UserCreationForm
from django.contrib.auth.models import User

class SignUpForm(UserCreationForm):
    email = forms.EmailField(required=True)
    class Meta:
        model = User
        fields = ("username", "email", "password1", "password2")

class StudentProfileForm(forms.ModelForm):
    class Meta:
        model = StudentProfile
        fields = ("admission_no", "date_of_birth", "batch")

4) Views
- Simple auth + profile edit views.

from django.shortcuts import render, redirect
from django.contrib.auth import login, authenticate
from .forms import SignUpForm, StudentProfileForm

def signup(request):
    if request.method == 'POST':
        form = SignUpForm(request.POST)
        if form.is_valid():
            user = form.save()
            login(request, user)
            return redirect('dashboard')
    else:
        form = SignUpForm()
    return render(request, 'accounts/signup.html', {'form': form})

def edit_profile(request):
    profile = request.user.studentprofile
    if request.method == 'POST':
        form = StudentProfileForm(request.POST, instance=profile)
        if form.is_valid():
            form.save()
            return redirect('profile')
    else:
        form = StudentProfileForm(instance=profile)
    return render(request, 'accounts/edit_profile.html', {'form': form})

10) Notes
- If you use a custom `User`, define it in `accounts/models.py` by subclassing `AbstractUser` and set `AUTH_USER_MODEL` in settings.
- Keep signals import in `apps.py` or `signals.py` and ensure the app config imports the signals on ready().
 
Project-specific models (from this repository)

- `User` (custom `AbstractUser`):
    - Uses `email` as `USERNAME_FIELD` and custom `CustomUserManager`.
    - `role` choices: `admin`, `teacher`, `student`, `guardian`.
    - Validations for `first_name`, `last_name`, `email`, and `phone_number`.
    - Helper properties: `is_admin`, `is_teacher`, `is_student`, `is_guardian`, and `full_name`.

- `Batch` and `BatchCourse`:
    - `Batch` fields: `name`, `year`, `start_date`, `end_date`, `is_active`, `created_by`.
    - `Batch.import_all_courses()` auto-imports active `Course` instances into the batch.
    - `BatchCourse` links `Batch` to `courses.Course` (unique per batch+course).

- `TeacherProfile`, `StudentProfile`, `GuardianProfile`:
    - `TeacherProfile` includes `teacher_id`, `designation`, `department`, `joining_date`, `subjects`, `photo`.
    - `StudentProfile` includes `student_id`, `date_of_birth`, `gender`, `nationality`, `batch`, `enrolled_courses` (through `StudentCourseEnrollment`), `emergency_contact`, `photo`.
    - `GuardianProfile` includes `guardian_id`, `relation_to_student`, `occupation`, `address`, `photo`.

- `StudentCourseEnrollment`:
    - Through model for `StudentProfile` ↔ `courses.Course`. Unique on `(student, course)` and records `enrollment_date`.

- `GuardianStudentRelationship`:
    - Links guardian `User` to student `User` with `is_primary` flag.
    - Model-level validation ensures guardian/student roles are correct.

Notes specific to this codebase
- Set `AUTH_USER_MODEL = 'accounts.User'` in your project `settings.py` before migrations.
- `accounts/signals.py` contains receivers that remove orphan guardian users when students are deleted; keep signal import wired in `apps.py`.
