Sent Emails / Email subsystem — Step-by-step guide

Overview
- Purpose: templates and record-keeping for emails sent by the system (e.g., registration, reports).
- Files: can include `utils/email.py`, `templates/email/`, and a `sent_emails` app or directory to store copies.

1) Basic email helper

from django.core.mail import EmailMultiAlternatives
from django.template.loader import render_to_string

def send_templated_email(subject, template_name, context, to_emails, html_template=None):
    text_body = render_to_string(template_name + '.txt', context)
    email = EmailMultiAlternatives(subject, text_body, None, to_emails)
    if html_template:
        html_body = render_to_string(html_template, context)
        email.attach_alternative(html_body, 'text/html')
    email.send()

2) Storing sent emails
- Create a `SentEmail` model to record subject, to, body, status, and timestamp.

from django.db import models

class SentEmail(models.Model):
    subject = models.CharField(max_length=255)
    to = models.TextField()  # comma separated
    body = models.TextField()
    sent_at = models.DateTimeField(auto_now_add=True)
    status = models.CharField(max_length=20)

3) Management and testing
- Use `django.core.mail.backends.console.EmailBackend` for dev and SMTP for production.
- For tests, use `django.core.mail.outbox` to assert emails.

4) Commands

```bash
# dev settings
export EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend
``` 

5) Notes
- Use `django-anymail` for multiple provider support (SendGrid, Mailgun, Amazon SES).
- Record failures and retries; consider Celery for async sending.
 
Project notes for this repository

- The repo does not include a centralized `sent_emails` Django app by default; create `utils/email.py` and an optional `sent_emails` app to track messages.

- Suggested model (in `sent_emails/models.py`):

from django.db import models

class SentEmail(models.Model):
    subject = models.CharField(max_length=255)
    to = models.TextField()
    body = models.TextField()
    sent_at = models.DateTimeField(auto_now_add=True)
    status = models.CharField(max_length=20)

- Use `EmailMultiAlternatives` for HTML + text emails, and queue sending via Celery in production.
- For local development, set `EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'`.
